The Cargo Book

cargo(1)

이름

cargo — 러스트 패키지 관리자

사용법

cargo [options] command [args]
cargo [options] --version
cargo [options] --list
cargo [options] --help
cargo [options] --explain code

설명

이 프로그램은 러스트 언어를 위한 패키지 관리자 및 빌드 도구이며, https://rust-lang.org에서 사용할 수 있습니다.

command may be one of:

  • built-in commands, see below
  • aliases
  • external tools

명령어

빌드 명령어

cargo-bench(1)
    패키지의 벤치마크를 실행합니다.

cargo-build(1)
    패키지를 컴파일합니다.

cargo-check(1)
    로컬 패키지와 모든 의존성의 오류를 확인합니다.

cargo-clean(1)
    Cargo가 이전에 생성한 결과물들을 제거합니다.

cargo-doc(1)
    패키지 문서를 빌드합니다.

cargo-fetch(1)
    네트워크에서 패키지 의존성을 가져옵니다.

cargo-fix(1)
    rustc가 보고한 린트 경고를 자동으로 수정합니다.

cargo-run(1)
    로컬 패키지의 바이너리나 예제를 실행합니다.

cargo-rustc(1)
    패키지를 컴파일하고 컴파일러에 추가 옵션을 전달합니다.

cargo-rustdoc(1)
    지정된 사용자 정의 플래그를 사용하여 패키지 문서를 빌드합니다.

cargo-test(1)
    패키지의 단위 및 통합 테스트를 실행합니다.

cargo-pkgid(1)
    정규화된 패키지 명세를 출력합니다.

cargo-remove(1)
    Cargo.toml 매니페스트 파일에서 의존성을 제거합니다.

cargo-tree(1)
    의존성 그래프를 트리 시각화로 표시합니다.

cargo-vendor(1)
    모든 의존성을 로컬로 벤더링합니다.

옵션

특별 옵션

-V
--version

Print version info and exit. If used with --verbose, prints extra information.

--list

List all installed Cargo subcommands. If used with --verbose, prints extra information.

--explain code

Run rustc --explain CODE which will print out a detailed explanation of an error message (for example, E0004).

표시 옵션

-v
--verbose

Use verbose output. May be specified twice for “very verbose” output which includes extra output such as dependency warnings and build script output. May also be specified with the term.verbose config value.

-q
--quiet

Do not print cargo log messages. May also be specified with the term.quiet config value.

--color when

Control when colored output is used. Valid values:

  • auto (default): Automatically detect if color support is available on the terminal.
  • always: Always display colors.
  • never: Never display colors.

May also be specified with the term.color config value.

매니페스트 옵션

--locked

Asserts that the exact same dependencies and versions are used as when the existing Cargo.lock file was originally generated. Cargo will exit with an error when either of the following scenarios arises:

  • The lock file is missing.
  • Cargo attempted to change the lock file due to a different dependency resolution.

It may be used in environments where deterministic builds are desired, such as in CI pipelines.

--offline

Prevents Cargo from accessing the network for any reason. Without this flag, Cargo will stop with an error if it needs to access the network and the network is not available. With this flag, Cargo will attempt to proceed without the network if possible.

Beware that this may result in different dependency resolution than online mode. Cargo will restrict itself to crates that are downloaded locally, even if there might be a newer version as indicated in the local copy of the index. See the cargo-fetch(1) command to download dependencies before going offline.

May also be specified with the net.offline config value.

--frozen

Equivalent to specifying both --locked and --offline.

공통 옵션

+toolchain

If Cargo has been installed with rustup, and the first argument to cargo begins with +, it will be interpreted as a rustup toolchain name (such as +stable or +nightly). See the rustup documentation for more information about how toolchain overrides work.

--config KEY=VALUE or PATH

Overrides a Cargo configuration value. The argument should be in TOML syntax of KEY=VALUE, or provided as a path to an extra configuration file. This flag may be specified multiple times. See the command-line overrides section for more information.

-C PATH

Changes the current working directory before executing any specified operations. This affects things like where cargo looks by default for the project manifest (Cargo.toml), as well as the directories searched for discovering .cargo/config.toml, for example. This option must appear before the command name, for example cargo -C path/to/my-project build.

This option is only available on the nightly channel and requires the -Z unstable-options flag to enable (see #10098).

-h
--help

Prints help information.

-Z flag

Unstable (nightly-only) flags to Cargo. Run cargo -Z help for details.

환경 변수

Cargo가 읽는 환경 변수에 대한 자세한 내용은 레퍼런스를 참조하세요.

종료 상태

  • 0: Cargo가 성공했습니다.
  • 101: Cargo가 완료하지 못했습니다.

파일

~/.cargo/
    다양한 파일을 저장하는 Cargo “홈” 디렉토리의 기본 위치입니다. 이 위치는 CARGO_HOME 환경 변수로 변경할 수 있습니다.

$CARGO_HOME/bin/
    cargo-install(1) 로 설치된 바이너리들이 이곳에 위치합니다. rustup 을 사용하는 경우, Rust와 함께 배포된 실행 파일들도 이곳에 위치합니다.

$CARGO_HOME/config.toml
    전역 설정 파일입니다. 설정 파일에 대한 자세한 내용은 레퍼런스를 참조하세요.

.cargo/config.toml
    Cargo는 현재 디렉토리와 모든 상위 디렉토리에서 .cargo/config.toml 이라는 파일을 자동으로 찾습니다. 이 설정 파일들은 전역 설정 파일과 병합됩니다.

$CARGO_HOME/credentials.toml
    레지스트리 로그인을 위한 개인 인증 정보입니다.

$CARGO_HOME/registry/
    이 디렉토리에는 레지스트리 인덱스의 캐시된 다운로드와 다운로드된 의존성들이 포함됩니다.

$CARGO_HOME/git/
    이 디렉토리에는 Git 의존성의 캐시된 다운로드들이 포함됩니다.

$CARGO_HOME 디렉토리의 내부 구조는 아직 안정적이지 않으며 변경될 수 있음에 유의하십시오.

예제

  1. 로컬 패키지와 그 모든 의존성을 빌드합니다:

    cargo build

  2. 최적화를 적용하여 패키지를 빌드합니다:

    cargo build --release

  3. 교차 컴파일된 타겟에 대해 테스트를 실행합니다:

    cargo test --target i686-unknown-linux-gnu

  4. 실행 파일을 빌드하는 새 패키지를 생성합니다:

    cargo new foobar

  5. 현재 디렉토리에 패키지를 생성합니다:

    mkdir foo && cd foo
cargo init .

  6. 명령어의 옵션과 사용법에 대해 알아봅니다:

    cargo help clean
