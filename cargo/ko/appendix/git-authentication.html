Git 인증

Cargo는 Git 의존성과 레지스트리를 사용할 때 몇 가지 형태의 인증을 지원합니다. 이 부록에는 Cargo와 함께 작동하는 방식으로 Git 인증을 설정하는 데 필요한 정보가 포함되어 있습니다.

다른 인증 방법이 필요한 경우, 내장 지원 대신 git 실행 파일을 사용하여 원격 저장소를 가져오도록 net.git-fetch-with-cli 설정 값을 지정할 수 있습니다. 이는 CARGO_NET_GIT_FETCH_WITH_CLI=true 환경 변수로 활성화할 수 있습니다.

Note: Cargo does not require authentication for public git dependencies so if you see an authentication failure in that context, ensure that the URL is correct.

HTTPS 인증

HTTPS 인증에는 credential.helper 메커니즘이 필요합니다. 여러 가지 자격 증명 헬퍼가 있으며, 글로벌 Git 설정 파일에서 사용할 헬퍼를 지정합니다.

# ~/.gitconfig

[credential]
helper = store

Cargo는 비밀번호를 묻지 않으므로, 대부분의 헬퍼의 경우 Cargo를 실행하기 전에 헬퍼에게 초기 사용자 이름/비밀번호를 제공해야 합니다. 한 가지 방법은 비공개 Git 저장소에 대해 git clone 을 실행하고 사용자 이름/비밀번호를 입력하는 것입니다.

팁:
macOS 사용자는 osxkeychain 헬퍼 사용을 고려해 보세요.
Windows 사용자는 GCM 헬퍼 사용을 고려해 보세요.

참고: Windows 사용자는 sh 쉘이 PATH 에 포함되어 있는지 확인해야 합니다. 이는 일반적으로 Git for Windows 설치 시 함께 제공됩니다.

SSH 인증

SSH 인증을 위해서는 SSH 키를 획득하기 위해 ssh-agent 가 실행 중이어야 합니다. 적절한 환경 변수(대부분의 Unix 계열 시스템에서는 SSH_AUTH_SOCK)가 설정되어 있는지, 그리고 올바른 키가 추가되었는지(ssh-add 사용) 확인하세요.

Windows는 Pageant(PuTTY 의 일부) 또는 ssh-agent 를 사용할 수 있습니다. ssh-agent 를 사용하려면 Cargo는 MinGW 또는 Cygwin에서 사용하는 시뮬레이션된 Unix 도메인 소켓을 지원하지 않으므로 Windows의 일부로 배포되는 OpenSSH를 사용해야 합니다. Windows 설치에 대한 자세한 내용은 Microsoft 설치 문서 에서 확인할 수 있으며, 키 관리 페이지에는 ssh-agent 를 시작하고 키를 추가하는 방법에 대한 지침이 있습니다.

참고: Cargo는 git@example.com:user/repo.git 과 같은 git의 단축 SSH URL을 지원하지 않습니다. ssh://git@example.com/user/repo.git 과 같은 전체 SSH URL을 사용하세요.

참고: SSH 설정 파일(OpenSSH의 ~/.ssh/config 등)은 Cargo의 내장 SSH 라이브러리에서 사용되지 않습니다. 더 복잡한 요구 사항이 있는 경우 net.git-fetch-with-cli 를 사용해야 합니다.

SSH 알려진 호스트 (Known Hosts)

SSH 호스트에 연결할 때 Cargo는 호스트 키 목록인 “known hosts“를 사용하여 호스트의 신원을 확인해야 합니다. Cargo는 표준 위치(홈 디렉토리의 .ssh/known_hosts, Unix 계열 플랫폼의 경우 /etc/ssh/ssh_known_hosts, Windows의 경우 %PROGRAMDATA%\ssh\ssh_known_hosts)에 있는 OpenSSH 스타일의 known_hosts 파일에서 이러한 알려진 호스트를 찾을 수 있습니다. 이 파일에 대한 자세한 내용은 sshd 맨 페이지 에서 확인할 수 있습니다. 또는 net.ssh.known-hosts 를 사용하여 Cargo 설정 파일에서 키를 구성할 수도 있습니다.

known hosts가 구성되기 전에 SSH 호스트에 연결하면 Cargo는 호스트 키를 추가하는 방법을 알려주는 오류 메시지를 표시합니다. 여기에는 시각적으로 확인하기 더 쉬운 호스트 키의 작은 해시인 “지문(fingerprint)“도 포함됩니다. 서버 관리자는 공개 키에 대해 ssh-keygen 을 실행하여(예: ssh-keygen -l -f /etc/ssh/ssh_host_ecdsa_key.pub) 지문을 얻을 수 있습니다. 잘 알려진 사이트는 웹에 지문을 게시할 수 있습니다. 예를 들어 GitHub는 https://docs.github.com/en/authentication/keeping-your-account-and-data-secure/githubs-ssh-key-fingerprints에 게시합니다.

Cargo에는 github.com 에 대한 호스트 키가 내장되어 있습니다. 만약 키가 변경되면 설정이나 known_hosts 파일에 새 키를 추가할 수 있습니다.

참고: Cargo는 known_hosts 파일의 @cert-authority 또는 @revoked 마커를 지원하지 않습니다. 이 기능을 사용하려면 net.git-fetch-with-cli 를 사용하세요. 이는 Cargo의 SSH 클라이언트가 예상대로 작동하지 않는 경우에도 좋은 팁입니다.
