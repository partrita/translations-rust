The Cargo Book

용어집

산출물 (Artifact)

산출물(artifact) 은 컴파일 프로세스의 결과로 생성된 파일 또는 파일 집합입니다. 여기에는 링크 가능한 라이브러리, 실행 가능한 바이너리 및 생성된 문서가 포함됩니다.

Cargo

Cargo 는 Rust 패키지 관리자 이며, 이 책의 주요 주제입니다.

Cargo.lock

락 파일(lock file) 을 참조하세요.

Cargo.toml

매니페스트(manifest) 를 참조하세요.

크레이트 (Crate)

러스트 크레이트(crate) 는 라이브러리 또는 실행 프로그램이며, 각각 라이브러리 크레이트 또는 바이너리 크레이트 라고 합니다.

Cargo 패키지 에 대해 정의된 모든 타겟 은 크레이트 입니다.

느슨하게 말해서, 크레이트 라는 용어는 타겟의 소스 코드나 타겟이 생성하는 컴파일된 산출물을 가리킬 수 있습니다. 또한 레지스트리 에서 가져온 압축된 패키지를 가리킬 수도 있습니다.

주어진 크레이트의 소스 코드는 다시 모듈(modules) 로 세분화될 수 있습니다.

에디션 (Edition)

러스트 에디션(Rust edition) 은 러스트 언어의 발전적 이정표입니다. 패키지의 에디션 은 Cargo.toml 매니페스트 에 지정되며, 개별 타겟은 자신이 사용하는 에디션을 지정할 수 있습니다. 자세한 내용은 에디션 가이드 를 참조하세요.

기능 (Feature)

기능(feature) 의 의미는 문맥에 따라 다릅니다:

  • 기능(feature) 은 조건부 컴파일을 가능하게 하는 이름이 지정된 플래그입니다. 기능은 선택적 의존성을 가리키거나, 소스 코드 내에서 확인할 수 있는 Cargo.toml 매니페스트 에 정의된 임의의 이름을 가리킬 수 있습니다.

  • Cargo에는 Cargo 자체의 실험적인 동작을 활성화하는 데 사용할 수 있는 불안정한 기능 플래그(unstable feature flags) 가 있습니다.

  • 러스트 컴파일러와 Rustdoc은 자체적인 불안정한 기능 플래그를 가지고 있습니다 (The Unstable Book 및 The Rustdoc Book 참조).

  • CPU 타겟에는 CPU의 기능을 지정하는 타겟 기능(target features) 이 있습니다.

인덱스 (Index)

인덱스(index) 는 레지스트리 에 있는 크레이트 의 검색 가능한 목록입니다.

락 파일 (Lock file)

Cargo.lock 락 파일(lock file) 은 _워크스페이스 _ 또는 _ 패키지_ 에서 사용되는 모든 의존성의 정확한 버전을 기록하는 파일입니다. 이 파일은 Cargo에 의해 자동으로 생성됩니다. Cargo.toml vs Cargo.lock 을 참조하세요.

매니페스트 (Manifest)

매니페스트(manifest) 는 Cargo.toml 이라는 파일에 작성된 패키지 또는 워크스페이스 에 대한 설명입니다.

가상 매니페스트(virtual manifest) 는 워크스페이스만을 설명하고 패키지를 포함하지 않는 Cargo.toml 파일입니다.

구성원 (Member)

구성원(member) 은 _워크스페이스 _ 에 속한 _ 패키지_ 입니다.

모듈 (Module)

러스트의 모듈 시스템은 코드를 모듈(modules) 이라고 불리는 논리적 단위로 구성하는 데 사용되며, 코드 내에서 격리된 네임스페이스를 제공합니다.

주어진 크레이트 의 소스 코드는 하나 이상의 별도 모듈로 세분화될 수 있습니다. 이는 일반적으로 코드를 관련 기능 영역별로 구성하거나, 소스 내 심볼(구조체, 함수 등)의 가시성 범위(공개/비공개)를 제어하기 위해 수행됩니다.

Cargo.toml 파일은 주로 정의하는 패키지, 해당 크레이트, 그리고 해당 크레이트가 의존하는 크레이트의 패키지와 관련이 있습니다. 그럼에도 불구하고 러스트로 작업할 때 “모듈“이라는 용어를 자주 접하게 되므로, 주어진 크레이트와의 관계를 이해해야 합니다.

패키지 (Package)

패키지(package) 는 소스 파일 모음과 패키지를 설명하는 Cargo.toml 매니페스트 파일의 집합입니다. 패키지에는 패키지 간의 의존성을 지정하는 데 사용되는 이름과 버전이 있습니다.

패키지에는 각각이 _크레이트 _ 인 여러 _ 타겟_ 이 포함되어 있습니다. Cargo.toml 파일은 패키지 내 크레이트의 유형(바이너리 또는 라이브러리)과 함께, 이 책 전반에서 설명하는 대로 각 크레이트가 어떻게 빌드되는지, 직접적인 의존성이 무엇인지 등 각 크레이트에 대한 메타데이터를 설명합니다.

패키지 루트(package root) 는 패키지의 Cargo.toml 매니페스트가 위치한 디렉토리입니다. (워크스페이스 루트 와 비교해 보세요.)

패키지 ID 명세(package ID specification) 또는 SPEC 은 특정 소스의 특정 패키지 버전을 고유하게 참조하는 데 사용되는 문자열입니다.

중소규모의 러스트 프로젝트는 하나의 패키지만 필요로 하는 경우가 많지만, 그 안에 여러 개의 크레이트를 포함하는 것은 흔한 일입니다.

규모가 더 큰 프로젝트는 여러 패키지를 포함할 수 있으며, 이 경우 Cargo 워크스페이스(workspaces) 를 사용하여 패키지 간의 공통 의존성 및 기타 관련 메타데이터를 관리할 수 있습니다.

패키지 관리자 (Package manager)

넓은 의미에서 패키지 관리자(package manager) 는 소프트웨어 생태계에서 산출물의 획득, 설치 및 업그레이드 프로세스를 자동화하는 프로그램(또는 관련 프로그램 모음)입니다. 프로그래밍 언어 생태계 내에서 패키지 관리자는 개발자 중심의 도구로서, 주요 기능은 중앙 저장소에서 라이브러리 산출물과 그 의존성을 다운로드하는 것입니다. 이 기능은 종종 (언어별 컴파일러를 호출하여) 소프트웨어 빌드를 수행하는 기능과 결합됩니다.

Cargo 는 러스트 생태계의 패키지 관리자입니다. Cargo는 러스트 패키지(#package) 의 의존성(_ 크레이트(#crate)_ 라고 불리는 _ 산출물(#artifact)_)을 다운로드하고, 패키지를 컴파일하고, 배포 가능한 패키지를 만들며, (선택적으로) 이를 러스트 커뮤니티의 _ 패키지 레지스트리(#registry)_ 인 crates.io 에 업로드합니다.

패키지 레지스트리 (Package registry)

레지스트리(registry) 를 참조하세요.

프로젝트 (Project)

패키지(#package) 의 또 다른 이름입니다.

레지스트리 (Registry)

레지스트리(registry) 는 설치하거나 _패키지(#package)_ 의 의존성으로 사용할 수 있는 다운로드 가능한 크레이트(#crate) 모음을 포함하는 서비스입니다. 러스트 생태계의 기본 레지스트리는 crates.io 입니다. 레지스트리는 모든 크레이트 목록을 포함하는 _ 인덱스(#index)_ 를 가지고 있으며, Cargo에게 필요한 크레이트를 다운로드하는 방법을 알려줍니다.

소스 (Source)

소스(source) 는 _패키지(#package)_ 의 의존성으로 포함될 수 있는 크레이트(#crate) 를 제공하는 제공자입니다. 소스에는 여러 종류가 있습니다:

  • 레지스트리 소스 (Registry source) — 레지스트리(#registry) 를 참조하세요.
  • 로컬 레지스트리 소스 (Local registry source) — 파일 시스템에 압축 파일로 저장된 크레이트 집합입니다. 로컬 레지스트리 소스 를 참조하세요.
  • 디렉토리 소스 (Directory source) — 파일 시스템에 압축되지 않은 파일로 저장된 크레이트 집합입니다. 디렉토리 소스 를 참조하세요.
  • 경로 소스 (Path source) — 파일 시스템에 위치한 개별 패키지(경로 의존성 등) 또는 여러 패키지 집합(경로 재정의 등)입니다.
  • Git 소스 (Git source) — Git 저장소에 위치한 패키지(Git 의존성 또는 Git 소스 등)입니다.

자체적인 자세한 내용은 소스 교체(Source Replacement) 를 참조하세요.

사양(Spec)

패키지 ID 명세(#package) 를 참조하세요.

타겟

타겟(target) 이라는 용어의 의미는 문맥에 따라 다릅니다:

  • Cargo 타겟 (Cargo Target) — Cargo 패키지(#package) 는 생성될 산출물(#artifact) 에 해당하는 타겟 들로 구성됩니다. 패키지는 라이브러리, 바이너리, 예제, 테스트 및 벤치마크 타겟을 가질 수 있습니다. 타겟 목록 은 Cargo.toml 매니페스트(#manifest) 에서 구성되며, 종종 소스 파일의 디렉토리 레이아웃 에 따라 자동으로 추론됩니다.

  • Target Directory — Cargo places built artifacts in the target directory. By default this is a directory named target at the workspace root, or the package root if not using a workspace. The directory may be changed with the --target-dir command-line option, the CARGO_TARGET_DIR environment variable, or the build.target-dir config option. For more information see the build cache documentation.

  • 타겟 아키텍처 (Target Architecture) — 빌드된 산출물이 실행될 OS 및 머신 아키텍처를 일반적으로 타겟 이라고 합니다.

  • 타겟 트리플 (Target Triple) — 트리플은 타겟 아키텍처를 지정하기 위한 특정 형식입니다. 트리플은 생성된 산출물의 아키텍처인 타겟 트리플 과 컴파일러가 실행 중인 아키텍처인 호스트 트리플 로 불릴 수 있습니다. 타겟 트리플은 --target 명령줄 옵션이나 build.target 설정 옵션 으로 지정할 수 있습니다. 트리플의 일반적인 형식은 <arch><sub>-<vendor>-<sys>-<abi> 이며 각 부분의 의미는 다음과 같습니다:

    • arch = 기본 CPU 아키텍처 (예: x86_64, i686, arm, thumb, mips 등)
    • sub = CPU 하위 아키텍처 (예: arm 의 경우 v7, v7s, v5te 등)
    • vendor = 벤더 (예: unknown, apple, pc, nvidia 등)
    • sys = 시스템 이름 (예: linux, windows, darwin 등). none 은 일반적으로 OS가 없는 베어메탈(bare-metal)에 사용됩니다.
    • abi = ABI (예: gnu, android, eabi 등) 일부 매개변수는 생략될 수 있습니다. 지원되는 타겟 목록을 보려면 rustc --print target-list 를 실행하세요.

테스트 타겟 (Test Targets)

Cargo 테스트 타겟(test targets) 은 코드의 올바른 작동과 정확성을 검증하는 데 도움이 되는 바이너리를 생성합니다. 테스트 산출물에는 두 가지 유형이 있습니다:

  • 단위 테스트 (Unit test) — 단위 테스트 는 라이브러리 또는 바이너리 타겟에서 직접 컴파일된 실행 바이너리입니다. 라이브러리 또는 바이너리 코드의 전체 내용을 포함하며, 개별 코드 단위를 검증하기 위해 #[test] 속성이 붙은 함수를 실행합니다.
  • 통합 테스트 타겟 (Integration test target) — _통합 테스트 타겟 _ 은 소스가 tests 디렉토리에 있거나 Cargo.toml _ 매니페스트 _ 의 [[test]] 테이블에 지정된 별개의 _ 크레이트_ 인 테스트 타겟 에서 컴파일된 실행 바이너리입니다. 이는 라이브러리의 공개 API만 테스트하거나, 바이너리를 실행하여 동작을 검증하기 위한 것입니다.

워크스페이스 (Workspace)

워크스페이스(workspace) 는 공통된 의존성 해결(공유된 Cargo.lock _ 락 파일 _ 사용), 출력 디렉토리, 그리고 프로파일과 같은 다양한 설정을 공유하는 하나 이상의 _ 패키지_ 모음입니다.

가상 워크스페이스(virtual workspace) 는 루트 Cargo.toml _ 매니페스트 _ 가 패키지를 정의하지 않고 워크스페이스 _ 구성원_ 만 나열하는 워크스페이스입니다.

워크스페이스 루트(workspace root) 는 워크스페이스의 Cargo.toml 매니페스트가 위치한 디렉토리입니다. (패키지 루트 와 비교해 보세요.)
