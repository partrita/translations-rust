The Cargo Book

환경 변수

Cargo는 사용자의 코드가 감지하거나 재정의할 수 있는 여러 환경 변수를 설정하고 읽습니다. 다음은 Cargo가 설정하는 변수 목록이며, 상호 작용 시점에 따라 정리되어 있습니다.

Cargo가 읽는 환경 변수

시스템에서 Cargo의 동작을 변경하기 위해 다음 환경 변수들을 재정의할 수 있습니다.

  • CARGO_LOG — Cargo는 디버그 로그 메시지를 표시하기 위해 tracing 크레이트를 사용합니다. CARGO_LOG 환경 변수를 trace, debug, warn과 같은 값으로 설정하여 디버그 로깅을 활성화할 수 있습니다. 일반적으로 디버깅 중에만 사용됩니다. 자세한 내용은 디버그 로깅(Debug logging)을 참조하세요.
  • CARGO_HOME — Cargo는 레지스트리 인덱스와 크레이트의 git 체크아웃에 대한 로컬 캐시를 유지합니다. 기본적으로 $HOME/.cargo(윈도우의 경우 %USERPROFILE%\.cargo)에 저장되지만, 이 변수를 통해 해당 디렉토리의 위치를 재정의할 수 있습니다. 한 번 캐시된 크레이트는 clean 명령어로 제거되지 않습니다. 자세한 내용은 가이드를 참조하세요.
  • CARGO_TARGET_DIR — 생성된 모든 결과물을 배치할 위치로, 현재 작업 디렉토리에 대한 상대 경로입니다. 설정을 통해 지정하려면 build.target-dir을 참조하세요.
  • CARGO — 이 변수가 설정되면, Cargo는 크레이트를 빌드하거나 빌드 스크립트 및 외부 하위 명령어를 실행할 때 자동으로 감지된 자체 경로 대신 이 값을 전달합니다. 이 값은 Cargo에 의해 직접 실행되지 않으며, 사용자가 기대하는 대로 항상 cargo와 똑같이 동작하는 명령을 가리켜야 합니다.
  • RUSTC — Cargo는 rustc를 직접 실행하는 대신 이 변수에 지정된 컴파일러를 실행합니다. 설정을 통해 지정하려면 build.rustc를 참조하세요.
  • RUSTC_WRAPPER — Cargo는 단순히 rustc를 실행하는 대신 이 변수에 지정된 래퍼를 실행합니다. 이때 rustc 호출 명령이 명령줄 인수로 전달되며, 첫 번째 인수는 실제 rustc의 경로가 됩니다. sccache와 같은 빌드 캐시 도구를 설정할 때 유용합니다. 설정을 통해 지정하려면 build.rustc-wrapper를 참조하세요. 이 값을 빈 문자열로 설정하면 설정을 덮어쓰고 래퍼를 사용하지 않도록 초기화합니다.
  • RUSTC_WORKSPACE_WRAPPER — Cargo는 단순히 rustc를 실행하는 대신, 워크스페이스 구성원에 대해 이 변수에 지정된 래퍼를 실행합니다. 이때 rustc 호출 명령이 명령줄 인수로 전달되며, 첫 번째 인수는 실제 rustc의 경로가 됩니다. 워크스페이스가 없는 단일 패키지 프로젝트를 빌드할 때 해당 패키지는 워크스페이스로 간주됩니다. 이는 파일 이름 해시에 영향을 주어 래퍼에 의해 생성된 결과물이 별도로 캐시되도록 합니다. 설정을 통해 지정하려면 build.rustc-workspace-wrapper를 참조하세요. 이 값을 빈 문자열로 설정하면 설정을 덮어쓰고 워크스페이스 구성원에 대해 래퍼를 사용하지 않도록 초기화합니다. RUSTC_WRAPPER와 RUSTC_WORKSPACE_WRAPPER가 모두 설정된 경우 중첩됩니다. 최종 호출은 $RUSTC_WRAPPER $RUSTC_WORKSPACE_WRAPPER $RUSTC가 됩니다.
  • RUSTDOC — Cargo는 rustdoc을 실행하는 대신 이 변수에 지정된 rustdoc 인스턴스를 실행합니다. 설정을 통해 지정하려면 build.rustdoc을 참조하세요.
  • RUSTDOCFLAGS — Cargo가 수행하는 모든 rustdoc 호출에 전달할 공백으로 구분된 사용자 정의 플래그 목록입니다. cargo rustdoc과 달리, 이는 모든 rustdoc 인스턴스에 플래그를 전달할 때 유용합니다. 플래그를 설정하는 다른 방법은 build.rustdocflags를 참조하세요. 이 문자열은 공백으로 나뉩니다. 여러 인수를 더 안정적으로 인코딩하려면 CARGO_ENCODED_RUSTDOCFLAGS를 참조하세요.
  • CARGO_ENCODED_RUSTDOCFLAGS — Cargo가 수행하는 모든 rustdoc 호출에 전달할 0x1f(ASCII Unit Separator)로 구분된 사용자 정의 플래그 목록입니다.
  • RUSTFLAGS — Cargo가 수행하는 모든 컴파일러 호출에 전달할 공백으로 구분된 사용자 정의 플래그 목록입니다. cargo rustc와 달리, 이는 모든 컴파일러 인스턴스에 플래그를 전달할 때 유용합니다. 플래그를 설정하는 다른 방법은 build.rustflags를 참조하세요. 이 문자열은 공백으로 나뉩니다. 여러 인수를 더 안정적으로 인코딩하려면 CARGO_ENCODED_RUSTFLAGS를 참조하세요.
  • CARGO_ENCODED_RUSTFLAGS — Cargo가 수행하는 모든 컴파일러 호출에 전달할 0x1f(ASCII Unit Separator)로 구분된 사용자 정의 플래그 목록입니다.
  • CARGO_INCREMENTAL — 이 값이 1로 설정되면 Cargo는 현재 컴파일에 대해 증분 컴파일(incremental compilation)을 강제로 활성화하고, 0으로 설정되면 강제로 비활성화합니다. 이 환경 변수가 없으면 Cargo의 기본값이 사용됩니다. build.incremental 설정 값도 참조하세요.
  • CARGO_CACHE_RUSTC_INFO — 이 값이 0으로 설정되면 Cargo는 컴파일러 버전 정보 캐싱을 시도하지 않습니다.
  • HTTPS_PROXY 또는 https_proxy 또는 http_proxy — 사용할 HTTP 프록시입니다. 자세한 내용은 http.proxy를 참조하세요.
  • HTTP_TIMEOUT — HTTP 타임아웃(초 단위)입니다. 자세한 내용은 http.timeout을 참조하세요.
  • TERM — 이 값이 dumb로 설정되면 진행률 표시줄이 비활성화됩니다.
  • BROWSER — The web browser to execute to open documentation with cargo doc’s --open flag, see doc.browser for more details.
  • RUSTFMT — cargo fmt은 rustfmt을 직접 실행하는 대신 이 변수에 지정된 rustfmt 인스턴스를 실행합니다.

설정 관련 환경 변수

Cargo는 일부 설정 값을 환경 변수에서 읽습니다. 자세한 내용은 설정(Configuration) 장을 참조하세요. 요약하자면, 지원되는 환경 변수는 다음과 같습니다.

  • CARGO_ALIAS_<이름> — 명령 별칭. alias를 참조하세요.
  • CARGO_BUILD_JOBS — 병렬 작업 수. build.jobs를 참조하세요.
  • CARGO_BUILD_RUSTC — rustc 실행 파일. build.rustc를 참조하세요.
  • CARGO_BUILD_RUSTC_WRAPPER — rustc 래퍼. build.rustc-wrapper를 참조하세요.
  • CARGO_BUILD_RUSTC_WORKSPACE_WRAPPER — 워크스페이스 구성원 전용 rustc 래퍼. build.rustc-workspace-wrapper를 참조하세요.
  • CARGO_BUILD_RUSTDOC — rustdoc 실행 파일. build.rustdoc를 참조하세요.
  • CARGO_BUILD_TARGET — 기본 타겟 플랫폼. build.target를 참조하세요.
  • CARGO_BUILD_TARGET_DIR — 기본 출력 디렉토리. build.target-dir를 참조하세요.
  • CARGO_BUILD_BUILD_DIR — The default build directory, see build.build-dir.
  • CARGO_BUILD_RUSTFLAGS — 추가 rustc 플래그. build.rustflags를 참조하세요.
  • CARGO_BUILD_RUSTDOCFLAGS — 추가 rustdoc 플래그. build.rustdocflags를 참조하세요.
  • CARGO_BUILD_INCREMENTAL — 증분 컴파일. build.incremental를 참조하세요.
  • CARGO_BUILD_DEP_INFO_BASEDIR — 의존성 정보 상대 디렉토리. build.dep-info-basedir를 참조하세요.
  • CARGO_CACHE_AUTO_CLEAN_FREQUENCY — Configures how often automatic cache cleaning runs, see cache.auto-clean-frequency.
  • CARGO_CARGO_NEW_VCS — cargo new 사용 시 기본 소스 제어 시스템. cargo-new.vcs를 참조하세요.
  • CARGO_FUTURE_INCOMPAT_REPORT_FREQUENCY — 미래 비호환성 보고서 알림 생성 빈도. future-incompat-report.frequency를 참조하세요.
  • CARGO_HTTP_DEBUG — HTTP 디버깅 활성화. http.debug를 참조하세요.
  • CARGO_HTTP_PROXY — HTTP 프록시 활성화. http.proxy를 참조하세요.
  • CARGO_HTTP_TIMEOUT — HTTP 타임아웃. http.timeout을 참조하세요.
  • CARGO_HTTP_CAINFO — TLS 인증서 인증 기관(CA) 파일. http.cainfo를 참조하세요.
  • CARGO_HTTP_PROXY_CAINFO — The proxy TLS certificate Certificate Authority file, see http.proxy-cainfo.
  • CARGO_HTTP_CHECK_REVOKE — TLS 인증서 폐기 확인 비활성화. http.check-revoke를 참조하세요.
  • CARGO_HTTP_SSL_VERSION — 사용할 TLS 버전. http.ssl-version을 참조하세요.
  • CARGO_HTTP_LOW_SPEED_LIMIT — HTTP 최저 속도 제한. http.low-speed-limit을 참조하세요.
  • CARGO_HTTP_MULTIPLEXING — HTTP/2 멀티플렉싱 사용 여부. http.multiplexing을 참조하세요.
  • CARGO_HTTP_USER_AGENT — HTTP user-agent 헤더. http.user-agent를 참조하세요.
  • CARGO_INSTALL_ROOT — cargo install의 기본 디렉토리. install.root를 참조하세요.
  • CARGO_NET_RETRY — 네트워크 오류 발생 시 재시도 횟수. net.retry를 참조하세요.
  • CARGO_NET_GIT_FETCH_WITH_CLI — 가져오기를 위해 git 실행 파일 사용 활성화. net.git-fetch-with-cli를 참조하세요.
  • CARGO_NET_OFFLINE — 오프라인 모드. net.offline을 참조하세요.
  • CARGO_PROFILE_<이름>_BUILD_OVERRIDE_<키> — 빌드 스크립트 프로파일 재정의. profile.<name>.build-override를 참조하세요.
  • CARGO_PROFILE_<이름>_CODEGEN_UNITS — 코드 생성 단위 설정. profile.<name>.codegen-units를 참조하세요.
  • CARGO_PROFILE_<이름>_DEBUG — 포함할 디버그 정보 종류. profile.<name>.debug를 참조하세요.
  • CARGO_PROFILE_<이름>_DEBUG_ASSERTIONS — 디버그 어설션 활성화/비활성화. profile.<name>.debug-assertions를 참조하세요.
  • CARGO_PROFILE_<이름>_INCREMENTAL — 증분 컴파일 활성화/비활성화. profile.<name>.incremental를 참조하세요.
  • CARGO_PROFILE_<이름>_LTO — 링크 타임 최적화(LTO). profile.<name>.lto를 참조하세요.
  • CARGO_PROFILE_<이름>_OVERFLOW_CHECKS — 오버플로 검사 활성화/비활성화. profile.<name>.overflow-checks를 참조하세요.
  • CARGO_PROFILE_<이름>_OPT_LEVEL — 최적화 레벨 설정. profile.<name>.opt-level를 참조하세요.
  • CARGO_PROFILE_<이름>_PANIC — 사용할 패닉 전략. profile.<name>.panic을 참조하세요.
  • CARGO_PROFILE_<이름>_RPATH — rpath 링크 옵션. profile.<name>.rpath를 참조하세요.
  • CARGO_PROFILE_<이름>_SPLIT_DEBUGINFO — 디버그 파일 출력 동작 제어. profile.<name>.split-debuginfo를 참조하세요.
  • CARGO_PROFILE_<이름>_STRIP — 심볼 및 디버그 정보 제거 제어. profile.<name>.strip을 참조하세요.
  • CARGO_REGISTRIES_<이름>_CREDENTIAL_PROVIDER — 레지스트리용 자격 증명 제공자. registries.<name>.credential-provider를 참조하세요.
  • CARGO_REGISTRIES_<이름>_INDEX — 레지스트리 인덱스 URL. registries.<name>.index를 참조하세요.
  • CARGO_REGISTRIES_<이름>_TOKEN — 레지스트리 인증 토큰. registries.<name>.token을 참조하세요.
  • CARGO_REGISTRY_CREDENTIAL_PROVIDER — crates.io용 자격 증명 제공자. registry.credential-provider를 참조하세요.
  • CARGO_REGISTRY_DEFAULT — --registry 플래그의 기본 레지스트리. registry.default를 참조하세요.
  • CARGO_REGISTRY_GLOBAL_CREDENTIAL_PROVIDERS — 특정 제공자가 정의되지 않은 레지스트리들을 위한 자격 증명 제공자 목록. registry.global-credential-providers를 참조하세요.
  • CARGO_REGISTRY_TOKEN — crates.io용 인증 토큰. registry.token을 참조하세요.
  • CARGO_TARGET_<triple>_LINKER — 사용할 링커. target.<triple>.linker를 참조하세요. 트리플(triple)은 대문자와 밑줄로 변환되어야 합니다.
  • CARGO_TARGET_<triple>_RUNNER — 실행 파일 러너. target.<triple>.runner를 참조하세요.
  • CARGO_TARGET_<triple>_RUSTFLAGS — 타겟용 추가 rustc 플래그. target.<triple>.rustflags를 참조하세요.
  • CARGO_TERM_QUIET — 저소음(quiet) 모드. term.quiet를 참조하세요.
  • CARGO_TERM_VERBOSE — 기본 터미널 상세 수준. term.verbose를 참조하세요.
  • CARGO_TERM_COLOR — 기본 색상 모드. term.color를 참조하세요.
  • CARGO_TERM_PROGRESS_WHEN — 기본 진행률 표시줄 모드. term.progress.when을 참조하세요.
  • CARGO_TERM_PROGRESS_WIDTH — 기본 진행률 표시줄 너비. term.progress.width를 참조하세요.

Cargo가 크레이트를 위해 설정하는 환경 변수

Cargo는 크레이트 컴파일 시 다음과 같은 환경 변수들을 노출합니다. 이는 cargo run 및 cargo test로 바이너리를 실행할 때도 적용됩니다. Rust 프로그램에서 이러한 변수 값을 가져오려면 다음과 같이 하세요.

let version = env!("CARGO_PKG_VERSION");

이제 version 변수에 CARGO_PKG_VERSION의 값이 포함됩니다.

매니페스트에 이러한 값 중 하나가 제공되지 않으면, 해당 환경 변수는 빈 문자열("")로 설정됩니다.

  • CARGO — 빌드를 수행하는 cargo 바이너리의 경로입니다.
  • CARGO_MANIFEST_DIR — 패키지의 매니페스트 파일이 들어있는 디렉토리입니다.
  • CARGO_MANIFEST_PATH — 패키지의 매니페스트 파일 경로입니다.
  • CARGO_PKG_VERSION — 패키지의 전체 버전입니다.
  • CARGO_PKG_VERSION_MAJOR — 패키지의 메이저(major) 버전입니다.
  • CARGO_PKG_VERSION_MINOR — 패키지의 마이너(minor) 버전입니다.
  • CARGO_PKG_VERSION_PATCH — 패키지의 패치(patch) 버전입니다.
  • CARGO_PKG_VERSION_PRE — 패키지의 프리릴리스(pre-release) 버전입니다.
  • CARGO_PKG_AUTHORS — 패키지 매니페스트에 기재된 저자 목록(콜론으로 구분됨)입니다.
  • CARGO_PKG_NAME — 패키지의 이름입니다.
  • CARGO_PKG_DESCRIPTION — 패키지 매니페스트에 기재된 설명입니다.
  • CARGO_PKG_HOMEPAGE — 패키지 매니페스트에 기재된 홈페이지 주소입니다.
  • CARGO_PKG_REPOSITORY — 패키지 매니페스트에 기재된 저장소(repository) 주소입니다.
  • CARGO_PKG_LICENSE — 패키지 매니페스트에 기재된 라이선스입니다.
  • CARGO_PKG_LICENSE_FILE — 패키지 매니페스트에 기재된 라이선스 파일 경로입니다.
  • CARGO_PKG_RUST_VERSION — 패키지 매니페스트에 기재된 Rust 버전입니다. 이는 현재 실행 중인 Rust 버전이 아니라 패키지가 지원하는 최소 Rust 버전임에 유의하세요.
  • CARGO_PKG_README — 패키지의 README 파일 경로입니다.
  • CARGO_CRATE_NAME — 현재 컴파일 중인 크레이트의 이름입니다. 라이브러리, 바이너리, 예제, 통합 테스트 또는 벤치마크와 같은 Cargo 타겟의 이름에서 -가 _로 변환된 형태입니다.
  • CARGO_BIN_NAME — 현재 컴파일 중인 바이너리의 이름입니다. 바이너리 또는 바이너리 예제에 대해서만 설정됩니다. 이 이름에는 .exe와 같은 파일 확장자가 포함되지 않습니다.
  • OUT_DIR — If the package has a build script, this is set to the folder where the build script should place its output. See below for more information. (Only set during compilation.) Cargo does not guarantee that this directory is empty, and it is not cleaned between builds.
  • CARGO_BIN_EXE_<name> — 바이너리 타겟 실행 파일의 절대 경로입니다. 이는 오직 통합 테스트 또는 벤치마크를 빌드할 때만 설정됩니다. 테스트 목적으로 실행할 파일을 찾기 위해 env 매크로와 함께 사용될 수 있습니다. <name>은 바이너리 타겟의 이름 그대로를 사용합니다. 예를 들어 이름이 my-program인 바이너리의 경우 CARGO_BIN_EXE_my-program이 됩니다. 테스트 빌드 시 바이너리도 자동으로 빌드되지만, 바이너리에 필요한 기능이 활성화되지 않은 경우는 제외됩니다.
  • CARGO_PRIMARY_PACKAGE — 빌드 중인 패키지가 ’주요 패키지(primary package)’인 경우 이 환경 변수가 설정됩니다. 주요 패키지란 사용자가 -p 플래그로 명령줄에서 선택했거나, 현재 디렉토리 및 기본 워크스페이스 구성원을 바탕으로 기본 선택된 패키지를 의미합니다. 의존성을 빌드할 때는 이 변수가 설정되지 않지만, 해당 의존성이 명령줄에서 함께 선택된 워크스페이스 구성원인 경우는 예외입니다. 이는 패키지 컴파일 시에만 설정됩니다 (바이너리나 테스트 실행 시에는 설정되지 않음).
  • CARGO_TARGET_TMPDIR — 오직 통합 테스트 또는 벤치마크 코드를 빌드할 때만 설정됩니다. 이는 타겟 디렉토리 내부의 경로로, 통합 테스트나 벤치마크에서 필요한 데이터를 자유롭게 보관할 수 있는 공간입니다. Cargo는 처음에 이 디렉토리를 생성만 할 뿐 그 내용을 관리하지는 않으며, 관리 책임은 테스트 코드에 있습니다.

동적 라이브러리 경로

Cargo는 cargo run이나 cargo test와 같은 명령으로 바이너리를 컴파일하고 실행할 때 동적 라이브러리 경로를 설정합니다. 이는 빌드 프로세스의 일부인 공유 라이브러리를 찾는 데 도움이 됩니다. 변수 이름은 플랫폼에 따라 다릅니다.

  • Windows: PATH
  • macOS: DYLD_FALLBACK_LIBRARY_PATH
  • Unix: LD_LIBRARY_PATH
  • AIX: LIBPATH

값은 Cargo가 시작될 때 기존 값에 덧붙여집니다. macOS의 경우 DYLD_FALLBACK_LIBRARY_PATH가 설정되어 있지 않으면 기본값인 $HOME/lib:/usr/local/lib:/usr/lib가 추가되는 특수한 처리가 있습니다.

Cargo는 다음 경로들을 포함합니다.

  • 빌드 스크립트에서 rustc-link-search 지시문을 통해 포함된 검색 경로들입니다. target 디렉토리 외부의 경로는 제거됩니다. 시스템상의 추가 라이브러리가 검색 경로에 필요한 경우 환경을 적절히 설정하는 것은 Cargo를 실행하는 사용자의 책임입니다.
  • target/debug와 같은 기본 출력 디렉토리 및 “deps” 디렉토리입니다. 이는 주로 절차적 매크로를 지원하기 위함입니다.
  • rustc sysroot 라이브러리 경로입니다. 이는 일반적으로 대부분의 사용자에게 중요하지 않습니다.

Cargo가 빌드 스크립트를 위해 설정하는 환경 변수

Cargo는 빌드 스크립트가 실행될 때 여러 환경 변수를 설정합니다. 이 변수들은 빌드 스크립트가 컴파일될 때는 아직 설정되지 않으므로, 앞서 본 env!를 사용하는 예제는 작동하지 않습니다. 대신 빌드 스크립트가 실행될 때 값을 가져와야 합니다.

use std::env;
let out_dir = env::var("OUT_DIR").unwrap();

이제 out_dir 변수에 OUT_DIR의 값이 포함됩니다.

  • CARGO — 빌드를 수행하는 cargo 바이너리의 경로입니다.

  • CARGO_MANIFEST_DIR — 빌드 중인 패키지(빌드 스크립트를 포함하는 패키지)의 매니페스트가 있는 디렉토리입니다. 또한 이는 빌드 스크립트가 시작될 때의 현재 작업 디렉토리 값이기도 합니다.

  • CARGO_MANIFEST_PATH — 패키지의 매니페스트 파일 경로입니다.

  • CARGO_MANIFEST_LINKS — 매니페스트의 links 값입니다.

  • CARGO_MAKEFLAGS — 하위 프로세스를 병렬화하기 위한 Cargo의 jobserver 구현에 필요한 매개변수를 포함합니다. build.rs에서 실행되는 rustc 또는 cargo 호출은 이미 CARGO_MAKEFLAGS를 읽을 수 있지만, GNU Make는 플래그를 인수로 직접 지정하거나 MAKEFLAGS 환경 변수를 통해 지정해야 합니다. 현재 Cargo는 MAKEFLAGS 변수를 직접 설정하지 않지만, GNU Make를 호출하는 빌드 스크립트에서 이 변수를 CARGO_MAKEFLAGS의 내용으로 자유롭게 설정할 수 있습니다.

  • CARGO_FEATURE_<이름> — 빌드 중인 패키지의 각 활성화된 기능에 대해 이 환경 변수가 생성됩니다. 여기서 <이름>은 기능 이름을 대문자로 바꾸고 -를 _로 변환한 것입니다.

  • CARGO_CFG_<cfg> — 빌드 중인 패키지의 각 설정 옵션에 대해, 이 환경 변수는 해당 설정 값을 포함합니다. 여기서 <cfg>는 설정 이름을 대문자로 바꾸고 -를 _로 변환한 것입니다. 불리언 설정은 설정된 경우에만 존재합니다. 여러 값을 갖는 설정은 ,로 구분되어 하나의 변수로 결합됩니다. 여기에는 컴파일러에 내장된 값(rustc --print=cfg로 확인 가능), 빌드 스크립트에 의해 설정된 값, 그리고 rustc에 전달된 추가 플래그(RUSTFLAGS 등)가 포함됩니다. 몇 가지 예시는 다음과 같습니다.

    • CARGO_CFG_FEATURE — 빌드 중인 패키지의 각 활성화된 기능입니다.
    • CARGO_CFG_UNIX — 유닉스 계열 플랫폼에서 설정됩니다.
    • CARGO_CFG_WINDOWS — 윈도우 계열 플랫폼에서 설정됩니다.
    • CARGO_CFG_TARGET_FAMILY=unix,wasm — 타겟 패밀리(target family)입니다.
    • CARGO_CFG_TARGET_OS=macos — 타겟 운영체제입니다.
    • CARGO_CFG_TARGET_ARCH=x86_64 — CPU 타겟 아키텍처입니다.
    • CARGO_CFG_TARGET_VENDOR=apple — 타겟 벤더입니다.
    • CARGO_CFG_TARGET_ENV=gnu — 타겟 환경 ABI입니다.
    • CARGO_CFG_TARGET_ABI=eabihf — The target ABI.
    • CARGO_CFG_TARGET_POINTER_WIDTH=64 — CPU 포인터 너비입니다.
    • CARGO_CFG_TARGET_ENDIAN=little — CPU 타겟 엔디언(endianness)입니다.
    • CARGO_CFG_TARGET_FEATURE=mmx,sse — 활성화된 CPU 타겟 기능 목록입니다.

    타겟 트리플마다 서로 다른 cfg 값 세트를 가지므로, 특정 타겟 트리플에 존재하는 변수가 다른 트리플에서는 사용 불가능할 수도 있습니다.

    Some cfg values like test are not available.

  • OUT_DIR — the folder in which all output and intermediate artifacts should be placed. This folder is inside the build directory for the package being built, and it is unique for the package in question. Cargo does not clean or reset this directory between builds, and its contents may persist across rebuilds. Build scripts should not assume that OUT_DIR is empty, and are responsible for managing or cleaning up any files they create.

  • TARGET — 컴파일 대상인 타겟 트리플입니다. 네이티브 코드는 이 트리플에 맞춰 컴파일되어야 합니다. 자세한 내용은 타겟 트리플 설명을 참조하세요.

  • HOST — Rust 컴파일러의 호스트 트리플입니다.

  • NUM_JOBS — 최상위 수준에서 지정된 병렬 처리 수준입니다. 이는 make와 같은 시스템에 -j 매개변수를 전달할 때 유용할 수 있습니다. 다만, 이 환경 변수를 해석할 때는 주의가 필요합니다. 역사적인 이유로 여전히 제공되지만, 최신 버전의 Cargo는 make -j를 실행할 필요가 없으며, 대신 MAKEFLAGS 환경 변수를 CARGO_MAKEFLAGS 내용으로 설정하여 하위 make 호출에 대해 Cargo의 GNU Make 호환 jobserver를 활성화할 수 있습니다.

  • DEBUG — true if any debug information will be generated and false otherwise.

  • OPT_LEVEL — values of the corresponding opt-level variable for the profile currently being built.

  • PROFILE — 릴리스 빌드의 경우 release, 그 외 빌드의 경우 debug입니다. 이는 프로파일이 dev를 상속하는지 release를 상속하는지에 따라 결정됩니다. 이 환경 변수를 사용하는 것은 권장되지 않습니다. OPT_LEVEL과 같은 다른 환경 변수를 사용하는 것이 실제 사용 중인 설정에 대한 더 정확한 정보를 제공합니다.

  • DEP_<links>_<key> — For more information about this set of environment variables, see build script documentation about links.

  • RUSTC, RUSTDOC — Cargo가 사용하기로 결정한 컴파일러 및 문서 생성기입니다. 빌드 스크립트에서도 사용할 수 있도록 전달됩니다.

  • RUSTC_WRAPPER — Cargo가 사용 중인 rustc 래퍼(있는 경우)입니다. build.rustc-wrapper를 참조하세요.

  • RUSTC_WORKSPACE_WRAPPER — Cargo가 워크스페이스 구성원을 위해 사용 중인 rustc 래퍼(있는 경우)입니다. build.rustc-workspace-wrapper를 참조하세요.

  • RUSTC_LINKER — (지정된 경우) 현재 타겟에 대해 Cargo가 사용하기로 결정한 링커 바이너리의 경로입니다. 링커는 .cargo/config.toml을 편집하여 변경할 수 있습니다. 자세한 내용은 Cargo 설정(Configuration) 문서를 참조하세요.

  • CARGO_ENCODED_RUSTFLAGS — Cargo가 rustc를 호출할 때 사용하는 추가 플래그들로, 0x1f(ASCII Unit Separator) 문자로 구분됩니다. build.rustflags를 참조하세요. Rust 1.55부터 RUSTFLAGS는 환경 변수에서 제거되었으므로, 스크립트에서는 대신 CARGO_ENCODED_RUSTFLAGS를 사용해야 합니다.

  • CARGO_PKG_<변수> — 패키지 정보 변수들로, 크레이트 빌드 시 제공되는 것들과 이름 및 값이 동일합니다.

Environment variables Cargo sets for cargo test

Cargo sets several environment variables when tests are run. You can retrieve the values when the tests are run:

use std::env;
let out_dir = env::var("CARGO_BIN_EXE_foo").unwrap();
  • CARGO_BIN_EXE_<name> — The absolute path to a binary target’s executable. This is only set when running an integration test or benchmark. The <name> is the name of the binary target, exactly as-is. For example, CARGO_BIN_EXE_my-program for a binary named my-program. Binaries are automatically built when the test is built, unless the binary has required features that are not enabled.

Cargo가 제3자 하위 명령어를 위해 설정하는 환경 변수

Cargo는 제3자 하위 명령어(즉, $PATH에 있는 cargo-foobar와 같은 이름의 프로그램)에 다음과 같은 환경 변수를 노출합니다.

  • CARGO — 빌드를 수행하는 cargo 바이너리의 경로입니다.
  • CARGO_MAKEFLAGS — 하위 프로세스를 병렬화하기 위한 Cargo의 jobserver 구현에 필요한 매개변수를 포함합니다. 이 변수는 Cargo가 jobserver의 존재를 감지했을 때만 설정됩니다.

환경에 대한 더 자세한 정보를 얻으려면 cargo metadata를 실행해 보세요.
