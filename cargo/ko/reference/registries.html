The Cargo Book

레지스트리

Cargo는 “레지스트리“로부터 크레이트를 설치하고 의존성을 가져옵니다. 기본 레지스트리는 crates.io입니다. 레지스트리는 사용 가능한 크레이트의 검색 가능한 목록인 “인덱스“를 포함합니다. 또한 레지스트리는 Cargo에서 직접 새로운 크레이트를 배포할 수 있도록 웹 API를 제공할 수도 있습니다.

참고: 기존 레지스트리를 미러링하거나 벤더링(vendoring)하는 데 관심이 있다면 소스 대체(Source Replacement)를 살펴보세요.

레지스트리 서버를 구현하는 경우, Cargo와 레지스트리 사이의 프로토콜에 대한 자세한 내용은 레지스트리 운영(Running a Registry)을 참조하세요.

인증이 필요한 레지스트리를 사용하는 경우 레지스트리 인증(Registry Authentication)을 참조하세요. 자격 증명 제공자(credential provider)를 구현하는 경우 자세한 내용은 자격 증명 제공자 프로토콜(Credential Provider Protocol)을 참조하세요.

대체 레지스트리 사용하기

crates.io 이외의 레지스트리를 사용하려면, 레지스트리의 이름과 인덱스 URL을 .cargo/config.toml 파일에 추가해야 합니다. registries 테이블은 각 레지스트리에 대한 키를 가집니다. 예시는 다음과 같습니다.

[registries]
my-registry = { index = "https://my-intranet:8080/git/index" }

index 키는 레지스트리 인덱스가 포함된 git 저장소의 URL이거나, sparse+ 접두사가 붙은 Cargo sparse 레지스트리 URL이어야 합니다.

그러면 크레이트는 Cargo.toml의 의존성 항목에 registry 키와 해당 레지스트리의 이름을 지정하여 다른 레지스트리의 크레이트에 의존할 수 있습니다.

# Sample Cargo.toml
[package]
name = "my-project"
version = "0.1.0"
edition = "2024"

[dependencies]
other-crate = { version = "1.0", registry = "my-registry" }

대부분의 설정값과 마찬가지로, 인덱스는 설정 파일 대신 환경 변수로 지정할 수 있습니다. 예를 들어, 다음과 같은 환경 변수를 설정하는 것은 설정 파일을 정의하는 것과 동일한 효과를 냅니다.

CARGO_REGISTRIES_MY_REGISTRY_INDEX=https://my-intranet:8080/git/index

참고: crates.io는 다른 레지스트리의 크레이트에 의존하는 패키지를 허용하지 않습니다.

대체 레지스트리에 배포하기

레지스트리가 웹 API 접근을 지원한다면, Cargo를 통해 패키지를 레지스트리에 직접 배포할 수 있습니다. cargo publish와 같은 몇몇 Cargo 명령어는 사용할 레지스트리를 지정하기 위해 --registry 커맨드 라인 플래그를 받습니다. 예를 들어, 현재 디렉토리의 패키지를 배포하려면 다음과 같이 합니다.

  1. cargo login --registry=my-registry

    이는 한 번만 수행하면 됩니다. 레지스트리 웹사이트에서 가져온 비밀 API 토큰을 입력해야 합니다. 또는 --token 커맨드 라인 플래그를 사용해 publish 명령어에 토큰을 직접 전달하거나, CARGO_REGISTRIES_MY_REGISTRY_TOKEN과 같이 레지스트리 이름이 포함된 환경 변수를 사용할 수도 있습니다.

  2. cargo publish --registry=my-registry

매번 --registry 커맨드 라인 옵션을 전달하는 대신, .cargo/config.toml의 registry.default 키를 사용하여 기본 레지스트리를 설정할 수 있습니다. 예를 들면 다음과 같습니다.

[registry]
default = "my-registry"

Cargo.toml 매니페스트에서 package.publish 키를 설정하면 패키지를 배포할 수 있는 레지스트리를 제한할 수 있습니다. 이는 소스코드가 공개되지 않은 패키지를 실수로 crates.io에 배포하는 것을 방지하는 데 유용합니다. 값은 레지스트리 이름의 리스트가 될 수 있습니다. 예시는 다음과 같습니다.

[package]
# ...
publish = ["my-registry"]

publish 값을 false로 설정하면 모든 배포가 제한되며, 이는 빈 리스트를 지정하는 것과 같습니다.

cargo login에 의해 저장된 인증 정보는 Cargo 홈 디렉토리(기본값 $HOME/.cargo)의 credentials.toml 파일에 저장됩니다. 각 레지스트리별로 별도의 테이블을 가집니다. 예시는 다음과 같습니다.

[registries.my-registry]
token = "854DvwSlUwEHtIo3kWy6x7UCPKHfzCmy"

레지스트리 프로토콜

Cargo는 git과 sparse 두 가지 원격 레지스트리 프로토콜을 지원합니다. 레지스트리 인덱스 URL이 sparse+로 시작하면 Cargo는 sparse 프로토콜을 사용합니다. 그렇지 않으면 Cargo는 git 프로토콜을 사용합니다.

git 프로토콜은 인덱스 메타데이터를 git 저장소에 저장하며, Cargo가 전체 저장소를 복제(clone)해야 합니다.

sparse 프로토콜은 일반 HTTP 요청을 사용하여 개별 메타데이터 파일을 가져옵니다. Cargo는 관련 있는 크레이트의 메타데이터만 다운로드하므로, sparse 프로토콜은 시간과 대역폭을 크게 절약할 수 있습니다.

crates.io 레지스트리는 두 프로토콜을 모두 지원합니다. crates.io에 대한 프로토콜은 registries.crates-io.protocol 설정 키를 통해 제어됩니다.
