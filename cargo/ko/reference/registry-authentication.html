레지스트리 인증

Cargo는 자격 증명 제공자(credential provider)를 통해 레지스트리에 인증합니다. 이 자격 증명 제공자들은 Cargo가 자격 증명을 저장하고 검색하는 데 사용하는 외부 실행 파일 또는 내장 제공자입니다.

인증이 필요한 대체 레지스트리를 사용하려면, 암호화되지 않은 자격 증명이 디스크에 저장되는 것을 방지하기 위해 자격 증명 제공자를 반드시 설정해야 합니다. 역사적인 이유로 공개(비인증) 레지스트리는 자격 증명 제공자 설정을 요구하지 않으며, 설정된 제공자가 없는 경우 cargo:token 제공자가 사용됩니다.

Cargo는 또한 운영 체제를 사용하여 토큰을 안전하게 저장하는 플랫폼별 제공자들을 포함하고 있습니다. credentials 파일에 암호화되지 않은 평문으로 자격 증명을 저장하는 cargo:token 제공자도 포함되어 있습니다.

권장 설정

$CARGO_HOME/config.toml에 전역 자격 증명 제공자 리스트를 설정하는 것이 권장되며, 기본값은 다음과 같습니다.

  • Windows: %USERPROFILE%\.cargo\config.toml
  • Unix: ~/.cargo/config.toml

이 권장 설정은 운영 체제 제공자를 사용하며, 실패할 경우 Cargo의 credentials 파일이나 환경 변수를 확인하는 cargo:token으로 폴백(fallback)합니다.

# ~/.cargo/config.toml
[registry]
global-credential-providers = ["cargo:token", "cargo:libsecret", "cargo:macos-keychain", "cargo:wincred"]

뒤에 있는 항목일수록 우선순위가 높음에 유의하세요. 자세한 내용은 registry.global-credential-providers를 참조하세요.

일부 프라이빗 레지스트리는 레지스트리 전용 자격 증명 제공자를 권장할 수도 있습니다. 사용하는 레지스트리의 문서를 확인하여 해당되는지 살펴보세요.

내장 제공자

Cargo는 여러 가지 내장 자격 증명 제공자를 포함하고 있습니다. 사용 가능한 내장 제공자는 향후 Cargo 릴리스에서 변경될 수 있습니다 (현재로서는 변경 계획이 없습니다).

cargo:token

Cargo의 credentials 파일을 사용하여 토큰을 암호화되지 않은 평문으로 저장합니다. 토큰을 검색할 때 CARGO_REGISTRIES_<NAME>_TOKEN 환경 변수를 확인합니다. 이 자격 증명 제공자가 리스트에 없으면 *_TOKEN 환경 변수는 작동하지 않습니다.

cargo:wincred

Windows 자격 증명 관리자(Credential Manager)를 사용하여 토큰을 저장합니다.

자격 증명은 자격 증명 관리자의 “Windows 자격 증명” 아래에 cargo-registry:<index-url>로 저장됩니다.

cargo:macos-keychain

macOS 키체인을 사용하여 토큰을 저장합니다.

키체인 접근(Keychain Access) 앱을 사용하여 저장된 토큰을 볼 수 있습니다.

cargo:libsecret

libsecret를 사용하여 토큰을 저장합니다.

libsecret을 지원하는 모든 비밀번호 관리자를 사용하여 저장된 토큰을 볼 수 있습니다. 다음은 몇 가지 예시입니다 (전체 목록은 아님):

  • 그놈 키링(GNOME Keyring)
  • KDE 지갑 관리자(KDE Wallet Manager) (KDE Frameworks 5.97.0부터)
  • KeePassXC (2.5.0부터)

cargo:token-from-stdout <command> <args>

표준 출력(stdout)으로 토큰을 반환하는 하위 프로세스를 실행합니다. 줄바꿈 문자는 제거됩니다.

  • 프로세스는 사용자의 표준 입력(stdin)과 표준 에러(stderr)를 상속받습니다.
  • 성공 시 0을 반환하고 종료해야 하며, 에러 발생 시 0이 아닌 값을 반환해야 합니다.
  • cargo login 및 cargo logout은 지원되지 않으며 사용 시 에러를 반환합니다.

실행되는 명령어에는 다음과 같은 환경 변수들이 제공됩니다.

  • CARGO — 명령어를 실행 중인 cargo 바이너리의 경로입니다.
  • CARGO_REGISTRY_INDEX_URL — 레지스트리 인덱스의 URL입니다.
  • CARGO_REGISTRY_NAME_OPT — (선택 사항) 레지스트리의 이름입니다. 검색 키(lookup key)로 사용되어서는 안 됩니다.

인자들은 하위 명령어로 그대로 전달됩니다.

자격 증명 플러그인

Cargo의 자격 증명 제공자 프로토콜(credential provider protocol)을 따르는 자격 증명 제공자 플러그인의 경우, 설정값은 실행 파일의 경로(또는 PATH에 있는 경우 실행 파일 이름) 문자열이어야 합니다.

예를 들어, crates.io에서 cargo-credential-1password를 설치하려면 다음과 같이 하세요.

cargo install cargo-credential-1password 명령어로 제공자를 설치합니다.

설정 파일에서 registry.global-credential-providers에 다음을 추가(또는 생성)합니다.

[registry]
global-credential-providers = ["cargo:token", "cargo-credential-1password --account my.1password.com"]

global-credential-providers에 들어가는 값들은 공백을 기준으로 경로와 커맨드 라인 인자로 나뉩니다. 경로 혹은 인자에 공백이 포함된 전역 자격 증명 제공자를 정의하려면, [credential-alias] 테이블을 사용하세요.
