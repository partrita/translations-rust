Box<[T]> 에 대한 IntoIterator 추가

요약

  • 박싱된 슬라이스(Boxed slices)는 모든 에디션에서 IntoIterator를 구현합니다.
  • 메서드 호출 문법(예: boxed_slice.into_iter())을 사용할 때, 2024년 이전 에디션에서는 IntoIterator::into_iter 호출이 _가려집니다 _. 따라서 boxed_slice.into_iter() 는 이전과 같이 여전히 (&(*boxed_slice)).into_iter() 로 해석됩니다.
  • Rust 2024에서 boxed_slice.into_iter() 는 IntoIterator::into_iter를 호출하는 것으로 의미가 변경됩니다.

세부 정보

Rust 1.80 이전까지는, 박싱된 슬라이스에 대해 IntoIterator 가 구현되지 않았습니다. 이전 버전에서는 박싱된 슬라이스에 대해 .into_iter() 를 호출하면, 메서드 호출이 자동으로 Box<[T]> 에서 &[T] 로 역참조하고, &T 참조를 생성하는 반복자를 반환했습니다. 예를 들어, 다음 코드는 이전 버전에서 작동했습니다:

#![allow(unused)]
fn main() {
// 이전 에디션에서의 동작 예시입니다.
let my_boxed_slice: Box<[u32]> = vec![1, 2, 3].into_boxed_slice();
// 참고: .into_iter()는 1.80 이전 버전에서 필수였습니다
for x in my_boxed_slice.into_iter() {
    // 2024 이전 에디션에서 x의 타입은 &u32입니다
}
}

Rust 1.80에서는 박싱된 슬라이스에 대한 IntoIterator 구현이 추가되었습니다. 이를 통해 참조 대신 값으로 슬라이스의 요소를 반복할 수 있습니다:

#![allow(unused)]
fn main() {
// 1.80부터 새로 추가됨, 모든 에디션 적용
let my_boxed_slice: Box<[u32]> = vec![1, 2, 3].into_boxed_slice();
for x in my_boxed_slice { // .into_iter()를 호출할 필요가 없음을 주목하세요
    // x의 타입은 u32입니다
}
}

이 예제는 모든 에디션에서 허용되는데, 이전에는 for 루프가 .into_iter() 메서드 호출처럼 자동으로 역참조하지 않았기 때문에 오류였던 코드입니다.

하지만 박싱된 슬라이스에 대해 수동으로 .into_iter() 를 호출하던 기존 코드가 참조에 대한 반복자에서 값에 대한 반복자로 변경되므로, 이는 일반적으로 호환성을 깨는 변경(breaking change)이 될 것입니다. 이 문제를 해결하기 위해, 박싱된 슬라이스에서의 .into_iter() 메서드 호출은 에디션에 따라 다르게 동작합니다. 2024 이전 에디션에서는 계속해서 참조에 대한 반복자를 반환하고, 2024 에디션부터는 값에 대한 반복자를 반환합니다.

#![allow(unused)]
fn main() {
// 2024 에디션에서 변경된 동작 예시
let my_boxed_slice: Box<[u32]> = vec![1, 2, 3].into_boxed_slice();
// 여전히 수동으로 .into_iter()를 호출하는 오래된 코드 예시
for x in my_boxed_slice.into_iter() {
    // 2024 에디션에서 x는 이제 u32 타입입니다
}
}

마이그레이션

boxed_slice_into_iter 린트는 참조를 반환하는 기존 동작을 유지하기 위해 박싱된 슬라이스에 대한 .into_iter() 호출을 .iter() 호출로 자동으로 수정합니다. 이 린트는 rust-2024-compatibility 린트 그룹의 일부이며, cargo fix --edition 을 실행할 때 자동으로 적용됩니다. 코드를 Rust 2024 에디션과 호환되도록 마이그레이션하려면 다음을 실행하세요:

cargo fix --edition

예를 들어, 이것은 다음과 같이 변경됩니다:

fn main() {
    let my_boxed_slice: Box<[u32]> = vec![1, 2, 3].into_boxed_slice();
    for x in my_boxed_slice.into_iter() {
        // x의 타입은 &u32입니다
    }
}

변경 후:

fn main() {
    let my_boxed_slice: Box<[u32]> = vec![1, 2, 3].into_boxed_slice();
    for x in my_boxed_slice.iter() {
        // x의 타입은 &u32입니다
    }
}

boxed_slice_into_iter 린트는 모든 에디션에서 경고하도록 기본 설정되어 있으므로, 수동으로 린트를 끄지 않았다면 마이그레이션 전에 이미 확인했을 것입니다.
