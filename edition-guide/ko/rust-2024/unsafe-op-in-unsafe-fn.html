unsafe_op_in_unsafe_fn 경고

요약

  • unsafe_op_in_unsafe_fn 린트가 이제 기본적으로 경고를 발생시킵니다. 이 경고는 unsafe 함수 내에서 명시적인 unsafe 블록 없이 수행되는 unsafe 연산 호출을 감지합니다.

세부 정보

unsafe_op_in_unsafe_fn 린트는 unsafe 함수 내에 명시적인 unsafe {} 블록 없이 unsafe 연산이 있을 경우 발생합니다.

#![allow(unused)]
fn main() {
#![warn(unsafe_op_in_unsafe_fn)]
unsafe fn get_unchecked<T>(x: &[T], i: usize) -> &T {
  x.get_unchecked(i) // WARNING: unsafe 블록이 필요함
}
}

해결 방법은 모든 unsafe 연산을 unsafe 블록으로 감싸는 것입니다:

#![allow(unused)]
fn main() {
#![deny(unsafe_op_in_unsafe_fn)]
unsafe fn get_unchecked<T>(x: &[T], i: usize) -> &T {
  unsafe { x.get_unchecked(i) }
}
}

이 변경 사항은 unsafe 함수 내에서 우발적으로 unsafe 연산을 사용하는 것을 방지하기 위한 것입니다. unsafe 함수 키워드는 두 가지 역할을 수행하고 있었습니다. 하나는 함수 호출 에 unsafe가 필요하며 호출자가 추가적인 안전 요구 사항을 준수할 책임이 있음을 선언하는 것이었습니다. 다른 하나는 함수 내부에서 unsafe 연산 사용을 허용하는 것이었습니다. 이 두 번째 역할은 명시적인 unsafe 블록 없이는 너무 위험하다고 판단되었습니다.

더 자세한 정보와 동기는 RFC #2585에서 확인할 수 있습니다.

마이그레이션

unsafe_op_in_unsafe_fn 린트는 rust-2024-compatibility 린트 그룹의 일부입니다. 코드를 Rust 2024 에디션과 호환되도록 마이그레이션하려면 다음을 실행하세요:

cargo fix --edition

대안으로, unsafe 블록 추가가 필요한 곳을 찾기 위해 수동으로 린트를 활성화하거나, 린트를 완전히 끄기 위해 allow로 전환할 수 있습니다.

#![allow(unused)]
fn main() {
// 수동 마이그레이션을 수행하려면 크레이트 루트에 이것을 추가하세요.
#![warn(unsafe_op_in_unsafe_fn)]
}
