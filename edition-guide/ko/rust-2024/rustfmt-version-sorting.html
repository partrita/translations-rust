Rustfmt: 버전 정렬

요약

rustfmt가 새로운 정렬 알고리즘을 사용합니다.

세부 정보

Rust 스타일 가이드에는 rustfmt가 임포트(imports)와 같은 다양한 컨텍스트에서 적용하는 정렬 규칙이 포함되어 있습니다.

이전 버전의 스타일 가이드와 Rustfmt는 일반적으로 "ASCIIbetical" 기반 접근 방식을 사용했습니다. 2024 에디션에서는 유니코드 문자를 사전순으로 비교하고 ASCII 숫자 비교에서 더 나은 결과를 제공하는 버전 정렬(version-sort) 유사 알고리즘을 사용하도록 변경되었습니다.

예를 들어 주어진 (정렬되지 않은) 입력에 대해:

use std::num::{NonZeroU32, NonZeroU16, NonZeroU8, NonZeroU64};
use std::io::{Write, Read, stdout, self};

이전 에디션에서는 rustfmt가 다음을 생성했을 것입니다:

use std::io::{self, stdout, Read, Write};
use std::num::{NonZeroU16, NonZeroU32, NonZeroU64, NonZeroU8};

2024 에디션에서 rustfmt는 이제 다음을 생성합니다:

use std::io::{self, Read, Write, stdout};
use std::num::{NonZeroU8, NonZeroU16, NonZeroU32, NonZeroU64};

마이그레이션

이 변경 사항은 2024 에디션으로 cargo fmt 또는 rustfmt를 실행하여 자동으로 적용할 수 있습니다. 마이그레이션 및 스타일 에디션 작동 방식에 대한 자세한 내용은 스타일 에디션 장을 참조하십시오.
