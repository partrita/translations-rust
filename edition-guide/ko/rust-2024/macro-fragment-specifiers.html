The Rust Edition Guide

매크로 조각 지정자

요약

  • expr 조각 지정자는 이제 const 및 _ 표현식도 지원합니다.
  • 하위 호환성을 위해 expr_2021 조각 지정자가 추가되었습니다.

세부 정보

Rust에 새로운 문법이 추가됨에 따라, 기존 macro_rules 조각 지정자는 하위 호환성을 유지하기 위해 때로는 새 문법과 매칭되는 것이 허용되지 않습니다. 이전 조각 지정자에서 새 문법을 지원하는 것은 다음 에디션까지 미뤄지기도 하며, 이때가 이를 업데이트할 기회를 제공합니다.

실제로 1.79에 추가된 const 표현식과 1.59에 추가된 _ 표현식에서 이런 일이 발생했습니다. 2021 에디션 및 그 이전 버전에서 expr 조각 지정자는 이러한 표현식과 매칭되지 않습니다. 이는 다음과 같은 시나리오가 있을 수 있기 때문입니다:

macro_rules! example {
    ($e:expr) => { println!("첫 번째 규칙"); };
    (const $e:expr) => { println!("두 번째 규칙"); };
}

fn main() {
    example!(const { 1 + 1 });
}

여기서, 2021 에디션에서는 매크로가 두 번째 규칙과 매칭됩니다. 만약 이전 에디션에서 expr이 새로 도입된 const 표현식과 매칭되도록 변경되었다면, 첫 번째 규칙과 매칭되었을 것이며, 이는 파괴적인 변경이 되었을 것입니다.

2024 에디션에서는 expr 지정자가 이제 const 및 _ 표현식과도 매칭됩니다. 이전 동작을 지원하기 위해, 새로운 표현식과 매칭되지 않는 expr_2021 조각 지정자가 추가되었습니다.

마이그레이션

edition_2024_expr_fragment_specifier 린트는 기존 매크로의 동작이 변경되지 않도록 expr 지정자의 모든 사용을 expr_2021로 변경합니다. 이 린트는 자동 에디션 마이그레이션에 포함된 rust-2024-compatibility 린트 그룹의 일부입니다. 코드를 Rust 2024 에디션과 호환되도록 마이그레이션하려면 다음을 실행하세요:

cargo fix --edition

대부분의 경우, 새로운 표현식을 지원하기 위해 대신 expr 지정자를 유지하고 싶을 것입니다. 매크로를 검토하여 const 또는 _와 매칭될 수 있는 다른 규칙이 있는지 확인하고 충돌이 있는지 판단해야 합니다. 새로운 동작을 원한다면 린트에 의해 변경된 내용을 되돌리면 됩니다.

대안으로, expr 지정자를 업데이트해야 할 수도 있는 매크로를 찾기 위해 수동으로 린트를 활성화할 수 있습니다.

#![allow(unused)]
fn main() {
// 수동 마이그레이션을 수행하려면 크레이트 루트에 이것을 추가하세요.
#![warn(edition_2024_expr_fragment_specifier)]
}
