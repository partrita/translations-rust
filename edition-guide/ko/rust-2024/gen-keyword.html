The Rust Edition Guide

gen 키워드

요약

  • gen 은 예약어 입니다.

세부 정보

gen 키워드는 Rust의 향후 릴리스에서 "gen 블록"을 도입하기 위해 RFC #3513 의 일부로 예약되었습니다. gen 블록은 특정 종류의 반복자를 더 쉽게 작성할 수 있는 방법을 제공할 것입니다. 지금 키워드를 예약하면 다음 에디션 전에 gen 블록을 안정화하기가 더 쉬워질 것입니다.

마이그레이션

gen 키워드를 도입하면 이미 gen 이라는 이름을 가진 식별자에 문제가 발생할 수 있습니다. 예를 들어, gen 이라는 이름의 변수나 함수는 새로운 키워드와 충돌합니다. 이를 극복하기 위해 Rust는 식별자가 키워드와 중복되는 것을 허용하는 원시 식별자(raw identifier) 를 위한 r# 접두사를 지원합니다.

keyword_idents_2024 린트는 코드가 두 에디션 모두에서 계속 작동하도록 gen 이라는 이름의 식별자를 r#gen 으로 자동 수정합니다. 이 린트는 rust-2024-compatibility 린트 그룹의 일부이며 cargo fix --edition 을 실행할 때 자동으로 적용됩니다. 코드를 Rust 2024 에디션과 호환되도록 마이그레이션하려면 다음을 실행하세요:

cargo fix --edition

예를 들어, 이것은 다음과 같이 변경됩니다:

fn gen() {
    println!("generating!");
}

fn main() {
    gen();
}

변경 후:

fn r#gen() {
    println!("generating!");
}

fn main() {
    r#gen();
}

대안으로, gen 식별자를 r#gen 으로 수정해야 하는 곳을 찾기 위해 수동으로 린트를 활성화할 수 있습니다:

#![allow(unused)]
fn main() {
// 수동 마이그레이션을 수행하려면 크레이트 루트에 이것을 추가하세요.
#![warn(keyword_idents_2024)]
}
