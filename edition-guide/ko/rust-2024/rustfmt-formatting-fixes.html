Rustfmt: 포매팅 수정

요약

  • 다양한 형식 지정 시나리오에 대한 수정 사항.

세부 정보

2024 스타일 에디션은 다양한 형식 지정 시나리오에 대한 여러 수정 사항을 도입합니다.

항목 뒤 또는 블록 끝에 관련 없는 후행 주석을 정렬하지 마십시오.

이전에는 rustfmt가 후행 주석이 있는 항목 뒤의 줄에 있는 주석이 후행 주석과 일치하도록 들여쓰기되어야 한다고 가정했습니다. 이것은 해당 주석이 들여쓰기되지 않도록 변경되었습니다.

스타일 에디션 2021:

pub const IFF_MULTICAST: ::c_int = 0x0000000800; // 멀티캐스트 지원
                                                 // 브로드캐스트 주소를 사용한 멀티캐스트.

pub const SQ_CRETAB: u16 = 0x000e; // 테이블 생성
pub const SQ_DRPTAB: u16 = 0x000f; // 테이블 삭제
pub const SQ_CREIDX: u16 = 0x0010; // 인덱스 생성
                                   //const SQ_DRPIDX: u16 = 0x0011; // 인덱스 삭제
                                   //const SQ_GRANT: u16 = 0x0012;  // 권한 부여
                                   //const SQ_REVOKE: u16 = 0x0013; // 권한 취소

fn foo() {
    let f = bar(); // Donec consequat mi. Quisque vitae dolor. Integer lobortis. Maecenas id nulla. Lorem.
// Id turpis. Nam posuere lectus vitae nibh. Etiam tortor orci, sagittis
// malesuada, rhoncus quis, hendrerit eget, libero. Quisque commodo nulla at
    let b = baz();

    let normalized = self.ctfont.all_traits().normalized_weight(); // [-1.0, 1.0]
                                                                   // TODO(emilio): 이 범위를 [.01, 10.0]으로 설정하여
                                                                   // css-fonts-4의 범위인 [1, 1000]에 맞추는 것이 합리적일 수 있습니다.
}

주석의 문자열을 들여쓰기하지 마십시오.

이전에는 rustfmt가 주석의 문자열을 잘못 형식화하려고 시도했습니다.

원본:

pub fn main() {
    /*   let s = String::from(
        "
hello
world
",
    ); */
}

스타일 에디션 2021:

pub fn main() {
    /*   let s = String::from(
            "
    hello
    world
    ",
        ); */
}

스타일 에디션 2024:

원본과 동일합니다.

긴 문자열은 표현식 형식 지정을 방해하지 않습니다.

일부 상황에서는 긴 문자열이 이전에 표현식의 형식 지정을 방해했습니다.

스타일 에디션 2021:

fn main() {
    let value = if x == "Excepteur sint occaecat cupidatat non proident, sunt in culpa qui officia deserunt mollit anim id est laborum." { 0 } else {10};

    let x = Testing {
              foo: "long_long_long_long_long_long_long_lo_long_long_long_long_long_long__long_long_long_long_long_long_",
bar: "long_long_long_long_long_long_long_long_long_long_lo_long_long_lolong_long_long_lo_long_long_lolong_long_long_lo_long_long_lo",
};
}

스타일 에디션 2024:

fn main() {
    let value = if x
        == "Excepteur sint occaecat cupidatat non proident, sunt in culpa qui officia deserunt mollit anim id est laborum."
    {
        0
    } else {
        10
    };

    let x = Testing {
        foo: "long_long_long_long_long_long_long_lo_long_long_long_long_long_long__long_long_long_long_long_long_",
        bar: "long_long_long_long_long_long_long_long_long_long_lo_long_long_lolong_long_long_lo_long_long_lolong_long_long_lo_long_long_lo",
    };
}

impl 블록에서 제네릭의 들여쓰기 수정

impl 항목의 제네릭이 과도하게 들여쓰기되었습니다.

스타일 에디션 2021:

impl<
        Target: FromEvent<A> + FromEvent<B>,
        A: Widget2<Ctx = C>,
        B: Widget2<Ctx = C>,
        C: for<'a> CtxFamily<'a>,
    > Widget2 for WidgetEventLifter<Target, A, B>
{
    type Ctx = C;
    type Event = Vec<Target>;
}

스타일 에디션 2024:

impl<
    Target: FromEvent<A> + FromEvent<B>,
    A: Widget2<Ctx = C>,
    B: Widget2<Ctx = C>,
    C: for<'a> CtxFamily<'a>,
> Widget2 for WidgetEventLifter<Target, A, B>
{
    type Ctx = C;
    type Event = Vec<Target>;
}

복잡한 fn 형식을 지정할 때 올바른 들여쓰기 사용

일부 경우, 복잡한 fn 서명이 비정상적인 들여쓰기로 끝날 수 있었으나 이제 수정되었습니다.

스타일 에디션 2021:

fn build_sorted_static_get_entry_names(
    mut entries: Vec<(u8, &'static str)>,
) -> (impl Fn(
    AlphabeticalTraversal,
    Box<dyn dirents_sink::Sink<AlphabeticalTraversal>>,
) -> BoxFuture<'static, Result<Box<dyn dirents_sink::Sealed>, Status>>
        + Send
        + Sync
        + 'static) {
}

스타일 에디션 2024:

fn build_sorted_static_get_entry_names(
    mut entries: Vec<(u8, &'static str)>,
) -> (
    impl Fn(
        AlphabeticalTraversal,
        Box<dyn dirents_sink::Sink<AlphabeticalTraversal>>,
    ) -> BoxFuture<'static, Result<Box<dyn dirents_sink::Sealed>, Status>>
    + Send
    + Sync
    + 'static
) {
}

중첩된 튜플 인덱싱 표현식에서 불필요한 공백 방지

중첩된 튜플 인덱싱 표현식에 불필요한 공백이 잘못 포함되곤 했습니다.

스타일 에디션 2021:

fn main() {
    let _ = ((1,),).0 .0;
}

스타일 에디션 2024:

fn main() {
    let _ = ((1,),).0.0;
}

match 내 블록 안의 return/break/continue를 세미콜론으로 끝내기

match 암(arm) 내 블록 안의 return, break, 또는 continue에 세미콜론이 잘못 누락되었습니다.

스타일 에디션 2021:

fn foo() {
    match 0 {
        0 => {
            return AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA
        }
        _ => "",
    };
}

스타일 에디션 2024:

fn foo() {
    match 0 {
        0 => {
            return AAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAAA;
        }
        _ => "",
    };
}

긴 배열 및 슬라이스 패턴이 이제 줄 바꿈(wrap)됩니다.

긴 배열 및 슬라이스 패턴이 제대로 줄 바꿈되지 않았습니다.

스타일 에디션 2021:

fn main() {
    let [aaaaaaaaaaaaaaaaaaaaaaaaaa, bbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbb, cccccccccccccccccccccccccc, ddddddddddddddddddddddddd] =
        panic!();
}

스타일 에디션 2024:

fn main() {
    let [
        aaaaaaaaaaaaaaaaaaaaaaaaaa,
        bbbbbbbbbbbbbbbbbbbbbbbbbbbbbbbb,
        cccccccccccccccccccccccccc,
        ddddddddddddddddddddddddd,
    ] = panic!();
}

마지막 표현식 구문을 표현식으로 형식 지정

블록의 마지막 구문이 표현식인 경우 이제 표현식으로 형식 지정됩니다.

스타일 에디션 2021:

fn main() {
    let toto = || {
        if true {
            42
        } else {
            24
        }
    };

    {
        T
    }
}

스타일 에디션 2024:

fn main() {
    let toto = || {
        if true { 42 } else { 24 }
    };

    { T }
}

함수 호출과 매크로 호출 간에 동일한 형식 지정

매크로 호출에서의 일부 형식 지정이 이제 함수 호출과 동일해졌습니다.

스타일 에디션 2021:

fn main() {
    macro_call!(HAYSTACK
        .par_iter()
        .find_any(|&&x| x[0] % 1000 == 999)
        .is_some());

    fn_call(
        HAYSTACK
            .par_iter()
            .find_any(|&&x| x[0] % 1000 == 999)
            .is_some(),
    );
}

스타일 에디션 2024:

fn main() {
    macro_call!(
        HAYSTACK
            .par_iter()
            .find_any(|&&x| x[0] % 1000 == 999)
            .is_some()
    );

    fn_call(
        HAYSTACK
            .par_iter()
            .find_any(|&&x| x[0] % 1000 == 999)
            .is_some(),
    );
}

단일 루프 본문을 가진 클로저에 대해 블록 클로저 강제

단일 루프를 가진 클로저는 이제 블록 표현식으로 형식 지정됩니다.

스타일 에디션 2021:

fn main() {
    thread::spawn(|| loop {
        println!("iteration");
    });
}

스타일 에디션 2024:

fn main() {
    thread::spawn(|| {
        loop {
            println!("iteration");
        }
    });
}

where 절의 빈 줄 제거됨

where 절의 빈 줄이 이제 제거됩니다.

스타일 에디션 2021:

fn foo<T>(_: T)
where
    T: std::fmt::Debug,

    T: std::fmt::Display,
{
}

스타일 에디션 2024:

fn foo<T>(_: T)
where
    T: std::fmt::Debug,
    T: std::fmt::Display,
{
}

속성이 있는 let-else 문의 형식 지정 수정

let-else 문에 속성이 있으면 else 절이 else 부분을 잘못 줄 바꿈하는 문제가 있었습니다.

스타일 에디션 2021:

fn main() {
    #[cfg(target_os = "linux")]
    let x = 42
    else {
        todo!()
    };

    // 이것은 속성이 없는 경우와 동일합니다.
    let x = 42 else { todo!() };
}

스타일 에디션 2024:

fn main() {
    #[cfg(target_os = "linux")]
    let x = 42 else { todo!() };

    // 이것은 속성이 없는 경우와 동일합니다.
    let x = 42 else { todo!() };
}

format_macro_matchers에 대한 1 차이(Off-by-one) 오류

format_macro_matchers 기능을 사용할 때, 매처(matcher)가 1칸 차이로 줄 바꿈되고 있었습니다.

스타일 에디션 2021:

macro_rules! test {
    ($aasdfghj:expr, $qwertyuiop:expr, $zxcvbnmasdfghjkl:expr, $aeiouaeiouaeio:expr, $add:expr) => {{
        return;
    }};
}

스타일 에디션 2024:

macro_rules! test {
    (
        $aasdfghj:expr, $qwertyuiop:expr, $zxcvbnmasdfghjkl:expr, $aeiouaeiouaeio:expr, $add:expr
    ) => {{
        return;
    }};
}

매치(match) => 뒤의 주석에 =>가 있을 때 발생하는 오류 수정

특정 상황에서 매치 표현식의 => 뒤에 있는 주석에 =>가 포함된 경우, 형식이 올바르게 지정되지 않는 오류가 발생했습니다.

스타일 에디션 2021:

fn main() {
    match a {
        _ =>
        // => 가 포함된 주석
                {
            println!("A")
        }
    }
}

스타일 에디션 2024:

fn main() {
    match a {
        _ =>
        // => 가 포함된 주석
        {
            println!("A")
        }
    }
}

매치 표현식의 여러 내부 속성이 잘못 들여쓰기됨

매치 표현식의 여러 내부 속성이 잘못 들여쓰기되고 있었습니다.

스타일 에디션 2021:

pub fn main() {
    match a {
        #![attr1]
    #![attr2]
        _ => None,
    }
}

스타일 에디션 2024:

pub fn main() {
    match a {
        #![attr1]
        #![attr2]
        _ => None,
    }
}
