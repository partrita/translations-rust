Cargo: 사용되지 않는 상속된 기본 기능 거부

요약

  • 워크스페이스 의존성이 default-features = true 를 지정하거나 (default-features를 지정하지 않은 경우) 상속된 워크스페이스 의존성에서 default-features = false를 사용하는 것은 더 이상 허용되지 않습니다.

세부 정보

워크스페이스 상속(Workspace inheritance)을 사용하면 의존성을 한 곳(워크스페이스)에 지정한 다음, 패키지 내에서 해당 워크스페이스 의존성을 참조할 수 있습니다. default-features 지정 방식에 의도치 않은 상호작용이 있었으며, 이는 2024 에디션에서 더 이상 허용되지 않습니다.

워크스페이스가 default-features = false 를 지정하지 않는 한, 상속된 패키지 의존성에서 default-features = false 를 지정하는 것은 더 이상 허용되지 않습니다. 예를 들어, 다음과 같이 지정된 워크스페이스가 있다고 가정해 보겠습니다:

[workspace.dependencies]
regex = "1.10.4"

다음은 이제 오류입니다:

[package]
name = "foo"
version = "1.0.0"
edition = "2024"

[dependencies]
regex = { workspace = true, default-features = false }  # ERROR

이 변경의 이유는 기본 기능(default features)이 이미 활성화된 상태에서 default-features = false를 지정할 때 발생하는 혼란을 피하기 위해서입니다. 왜냐하면 이는 아무런 효과가 없기 때문입니다.

의존성이 해당 의존성의 기본 기능을 활성화할지 여부를 결정하는 유연성을 원한다면, 워크스페이스 정의에서 default-features = false 로 설정해야 합니다. 단, 여러 워크스페이스 멤버를 동시에 빌드하는 경우 기능(feature)들이 통합되므로, 한 멤버가 default-features = true 로 설정하면(명시적으로 설정하지 않은 경우 이것이 기본값임) 해당 의존성을 사용하는 모든 멤버에 대해 기본 기능이 활성화된다는 점에 유의하세요.

마이그레이션

cargo fix --edition 을 사용할 때, Cargo는 Cargo.toml 파일에서 이러한 상황의 default-features = false 를 자동으로 제거합니다.

If you prefer to update your Cargo.toml manually, check for any warnings when running a build and remove the corresponding entries. Previous editions should display something like:

warning: /home/project/Cargo.toml: `default-features` is ignored for regex,
since `default-features` was not specified for `workspace.dependencies.regex`,
this could become a hard error in the future
